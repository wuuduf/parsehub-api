# 用户解析页面

服务启动后，直接访问站点根地址：

```text
https://你的域名/
```

## 使用流程

1. 首次访问点击右上角“API Key”，输入管理员面板生成的用户 Key。
2. 默认只保存在当前浏览器会话；勾选“保存在此设备”后写入 `localStorage`。
3. 粘贴作品链接或完整分享文案，点击“开始解析”。
4. 页面显示平台、标题、正文、原始页面和媒体预览。
5. 图片、视频和 Live Photo 可以分别勾选。
6. 视频存在多个真实可播放格式时，可以选择清晰度；格式来自上游解析结果，不会伪造不存在的画质。
7. “下载所选”下载当前勾选项目，“打包下载”由服务端创建异步 ZIP 任务。

## 视频质量说明

- YouTube、Facebook、Snapchat 等由 `yt-dlp` 返回格式列表的平台，页面会展示同时包含视频和音频、可直接由浏览器播放的格式。
- 相同分辨率存在多个格式时，服务端优先保留码率更高的一个。
- 纯视频 DASH 流需要音视频合并，当前不会作为浏览器清晰度选项，避免下载后没有声音。
- 平台解析器只返回一个媒体地址时，页面显示“默认”或该地址自带的分辨率。
- 视频地址通过 ParseHub 短期签名媒体网关交付，支持 `HEAD` 和 `Range`。

## API Key 安全

- 页面不会内置公共 API Key，每个用户应使用管理员单独生成的 Key。
- 公共或不可信设备不要勾选“保存在此设备”。
- 管理员可以在 `/admin` 立即停用或删除 Key。
- 生产站点必须使用 HTTPS，否则浏览器存储和请求中的 Key 可能被窃取。

## 浏览器下载限制

浏览器可能阻止一次自动下载多个文件。遇到提示时选择“允许此网站下载多个文件”，或使用“打包下载”获取一个 ZIP。
